# Code of Conduct

Orrery is an open-source engineering project. The standard here is simple: keep the work useful,
focused, and safe for people who are contributing code, testing the app, reporting issues, or using it
with their own data.

## Working Standard

- Discuss code, design, security, and product tradeoffs directly, but do not make it personal.
- Back technical claims with reproduction steps, logs, screenshots, tests, or clear reasoning.
- Keep issues and pull requests focused on one topic where possible.
- Respect maintainers' time: read the README, search existing issues, and include enough context for
  someone else to reproduce or review the problem.
- Be clear when something touches privacy, provider accounts, credentials, database access, generated
  files, or sandbox execution.
- Accept review as part of the work. Strong disagreement is fine; personal attacks are not.

## Unacceptable Behavior

The following is not allowed in Orrery project spaces:

- Harassment, threats, hate speech, sexualized comments, or targeted abuse.
- Personal insults, repeated bad-faith arguing, trolling, or derailing discussions after being asked to
  stop.
- Publishing or requesting private information, credentials, API keys, account tokens, database URLs,
  or other secrets.
- Encouraging unsafe integrations, credential theft, browser-session scraping, provider-account abuse,
  or bypassing security controls.
- Spamming issues, pull requests, discussions, or release pages.
- Using the project to attack, impersonate, or mislead other people or organizations.

## Security And Privacy

Do not post real secrets or private user data in public issues, pull requests, screenshots, logs, or
generated files. Redact anything sensitive before sharing it.

Security vulnerabilities must be reported privately through the process in [`SECURITY.md`](SECURITY.md).
Do not open a public issue for exploitable bugs, leaked secrets, authentication bypasses, sandbox
escapes, or anything that could put users' data at risk.

## Scope

This code applies to all Orrery project spaces, including GitHub issues, pull requests, discussions,
release comments, documentation, community channels, and public communication where someone is clearly
representing the project.

## Enforcement

Maintainers may remove comments, edit issue titles for clarity, close or lock threads, reject pull
requests, block accounts, or take other reasonable action when behavior harms the project or its
contributors.

Enforcement is based on behavior and impact, not seniority, contribution history, or technical skill.
If a maintainer asks you to move a discussion, stop a behavior, or provide safer reproduction details,
follow that direction.
