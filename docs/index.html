Current product

Live capabilities, labeled honestly.

Chat, data, dashboards, files, skills, and bounded agents work end to end. Automations and Media remain visible work in progress and are not presented as finished integrations.

Chat

Stream from connected local or cloud models, retrieve documents, and use only the tools enabled for the current message.

Data & Ontology

Connect PostgreSQL, browse read-only schemas, and build hybrid-search collections.

Dashboards

Create and revise saved dashboard specs, then refresh them from live data without another model call.

Files & Skills

Generate validated deliverables and guide work with reusable instruction playbooks.

Agents

Run scoped agents manually or on a schedule with budgets, durable traces, cancellation, and approval pauses.

In progress

Automation APIs and editor wiring, Media providers and library, and inbound Agent API/Slack/Gmail receivers are planned work.

Security model

Private by default, with gaps kept visible.

Orrery binds locally, stores supported secrets in the operating-system keychain, and treats model, document, and tool output as untrusted input.

Cloud-model calls use routes you select. Third-party web search is off per message until you explicitly enable it. Remaining approval and fail-closed work is listed in the public TODO.

Keychain secrets

Provider keys and supported connection secrets use the OS credential store.

Offline sandbox

Python, shell, generated-code work, and PDF OCR require the current bounded no-network image.

Read-only data

Connected-data queries are parsed, bounded, and run through read-only paths.

Honest backlog

Central Chat approvals, outage-safe authorization, redirect SSRF, and URL-secret handling remain P0 work.
